> 최종 갱신: 2026-03-18

---

## 1. 개요

### 목적

사용자의 월간/주간 지출 현황을 종합적으로 시각화하고, 카드별·카테고리별·태그별 통계와 태그 교차 분석을 제공하는 대시보드 기능을 정의한다. 가계부 데이터를 기반으로 지출 패턴을 파악하고, 혜택 절약 금액, 실적 진행률 등 핵심 지표를 한눈에 확인할 수 있다.

그룹 가계부 선택 시 그룹 레벨 통계(멤버별 지출 비교, 그룹 카테고리 분포 등)를 제공한다. 태그 교차 분석은 TAG-system.md의 통계 화면과 연동되어 다차원 인사이트를 제공한다.

### 대상 사용자

- 월간 지출 현황을 한눈에 파악하고 싶은 회원
- 카드별 실적 진행률을 확인하고 싶은 회원
- 카테고리별 지출 비율을 분석하고 싶은 회원
- 태그 기반 심층 분석을 하고 싶은 회원
- 그룹 가계부의 통계를 확인하고 싶은 그룹 멤버

## 3. 화면 명세

### 3.1 대시보드 메인 화면

- **진입 경로**: 홈 탭 (기본 화면), `/dashboard`

| 요소명 | 유형 | 필수 | 설명 |
|--------|------|------|------|
| 기간 선택기 | SegmentedControl + DatePicker | - | "이번 주" / "이번 달" / "커스텀" |
| 가계부 선택 | Select | - | "개인" (기본) 또는 그룹 선택 |
| 월간 요약 카드 | SummaryCard (4열) | - | 총 지출 / 결제 건수 / 절약 금액 / 전월 대비 |
| 카드별 실적 섹션 | Section | - | 카드별 지출 + ProgressBar |
| 카테고리 차트 | PieChart + Legend | - | 상위 8개 카테고리 + "기타" |
| 태그 통계 섹션 | Section | - | 상위 5개 태그 + "더보기" 링크 |
| 주간 추이 차트 | BarChart | - | 최근 4주 또는 12개월 지출 추이 |

### 3.2 월간 요약 카드

| 지표 | 표시 형식 | 색상 규칙 |
|------|----------|----------|
| 총 지출 | ₩1,234,567 | - |
| 결제 건수 | 42건 | - |
| 혜택 절약 | ₩23,400 | 초록색 |
| 전월 대비 | ▲12% 또는 ▼8% | 증가: 빨강, 감소: 초록 |

### 3.3 카드별 실적 섹션

| 요소명 | 유형 | 설명 |
|--------|------|------|
| 카드 이미지 + 이름 | Thumbnail + Text | 카드 식별 |
| 이번 달 지출 | Text (bold) | ₩500,000 |
| 실적 ProgressBar | ProgressBar (color) | 구간별 색상 (미달: 회색, 1구간: 파랑, 2구간: 초록) |
| 다음 구간까지 | Text (muted) | "다음 구간까지 ₩200,000 남음" |

### 3.4 카테고리별 통계

| 요소명 | 유형 | 설명 |
|--------|------|------|
| 파이 차트 | PieChart (interactive) | 카테고리별 비율, 클릭 시 상세 |
| 범례 | Legend | 카테고리명 + 금액 + 비율 |
| 상세 목록 | Expandable List | 각 카테고리의 결제 건 목록 |

### 3.5 태그 통계 섹션

| 요소명 | 유형 | 설명 |
|--------|------|------|
| 태그 랭킹 | HorizontalBarChart | 상위 5개 태그 지출 금액 |
| 더보기 링크 | TextLink | → `/dashboard/tags` (TAG 통계 화면) |
| 교차 분석 바로가기 | Button | → `/dashboard/tags/cross` |

### 3.6 그룹 대시보드 (그룹 선택 시)

## 4. Acceptance Criteria

| # | Given | When | Then |
|---|-------|------|------|
| AC-01 | 대시보드 진입 시 | 이번 달 데이터가 로드되고 | 월간 요약 카드에 총 지출, 건수, 절약 금액이 표시된다 |
| AC-02 | 기간을 "이번 주"로 변경하면 | 모든 통계가 | 이번 주 기준으로 갱신된다 |
| AC-03 | 카드별 실적 섹션에서 | 각 카드의 지출과 ProgressBar가 | 현재 실적 구간을 정확히 반영한다 |
| AC-04 | 카테고리 파이 차트에서 | 카테고리를 클릭하면 | 해당 카테고리의 결제 목록이 펼쳐진다 |
| AC-05 | 태그 통계 "더보기"를 클릭하면 | TAG 통계 상세 화면으로 | 이동한다 |
| AC-06 | 그룹을 선택하면 | 대시보드가 그룹 모드로 전환되고 | 멤버별 지출 비교가 표시된다 |
| AC-07 | 전월 대비 지출이 증가했으면 | 빨간 ▲ 아이콘과 | 증가 퍼센트가 표시된다 |
| AC-08 | 결제 데이터가 없는 기간을 선택하면 | EmptyState가 표시되고 | "결제 내역이 없습니다" 안내가 표시된다 |

## 5. Edge Cases & 에러 시나리오

| 시나리오 | 조건 | 처리 |
|----------|------|------|
| 신규 사용자 | 결제 데이터 0건 | 온보딩 카드 표시: "첫 결제를 입력해보세요" |
| 카드 미등록 | 등록된 카드 0장 | 카드 등록 유도 카드 표시 |
| 대량 데이터 | 월간 결제 500건 이상 | 집계 테이블(user_monthly_summary 등) 활용, 차트는 상위 N개만 |
| 데이터 갱신 지연 | 집계 테이블이 아직 갱신되지 않은 경우 | "마지막 갱신: 2시간 전" 표시 |
| 그룹 권한 | MEMBER가 그룹 대시보드 접근 | 본인+전체 통계 모두 조회 가능 (거버넌스상 조회는 전체 허용) |
| 환율 변동 | 해외 결제의 KRW 환산 금액 | 결제 시점의 환율 적용 (재계산 없음) |

## 7. 데이터 모델 연동

### 관련 테이블

| 테이블 | 역할 |
|--------|------|
| `user_monthly_summary` | 사용자 월간 종합 집계 |
| `user_card_monthly_summary` | 카드별 월간 집계 |
| `user_category_summary` | 카테고리별 집계 |
| `user_tag_summary` | 태그별 집계 |
| `user_performance` | 실적 현황 (ProgressBar) |
| `performance_tier` | 실적 구간 정의 |
| `payment` / `payment_item` | 원본 결제 데이터 (드릴다운용) |
| `payment_item_tag` / `tag` | 태그 교차 분석 |
| `ledger_group` / `group_member` | 그룹 대시보드 |

### 핵심 비즈니스 로직

1. **월간 요약**: `user_monthly_summary` 테이블에서 조회 (PaymentCreatedEvent 시 갱신)
2. **전월 대비**: 이번 달 vs 지난 달 `user_monthly_summary` 비교
3. **절약 금액**: payment_item에서 적용된 benefit_amount 합계
4. **카드별 실적**: `user_card_monthly_summary` + `user_performance` JOIN
5. **교차 분석**: MVP에서는 실시간 JOIN 쿼리 (TAG-system.md 참조)
6. **그룹 통계**: `payment WHERE group_id = :groupId` 기반 집계
