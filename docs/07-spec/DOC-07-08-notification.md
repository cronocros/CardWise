# CardWise 알림 서비스 상세 명세 (DOC-07-08)

> 최종 갱신: 2026-03-18

---

## 1. 개요

### 목적

CardWise 내 주요 이벤트(바우처 만료, 실적 구간 변동, 그룹 초대 등)에 대한 인앱 알림을 제공하는 기능을 정의한다. 사용자가 혜택을 놓치지 않도록 적시에 알림을 전달하고, 알림 유형별 on/off 설정을 지원한다.

MVP에서는 인앱 알림만 지원하며, 푸시 알림(FCM/APNs)은 Phase 2 모바일 앱 개발 시 추가한다. 알림은 도메인 이벤트(Spring @EventListener)에 의해 자동 생성된다.

### 대상 사용자

- 바우처 만료를 놓치지 않고 사용하고 싶은 회원
- 실적 달성/미달 현황을 파악하고 싶은 회원
- 그룹 가계부에 초대받은 회원
- 알림을 선택적으로 관리하고 싶은 회원

## 3. 화면 명세

### 3.1 알림 센터 화면

- **진입 경로**: 상단 네비게이션 → 벨 아이콘, `/notifications`

| 요소명 | 유형 | 필수 | 검증 규칙 |
|--------|------|------|----------|
| 벨 아이콘 + 배지 | IconButton + Badge | - | 미읽은 알림 수 표시 (99+) |
| 필터 탭 | TabGroup | - | "전체", "바우처", "실적", "그룹" |
| 전체 읽음 버튼 | TextButton | - | "모두 읽음 처리" |
| 알림 목록 | List (infinite scroll) | - | 최신순, 커서 페이지네이션 |
| 빈 상태 | EmptyState | - | "새로운 알림이 없습니다" |

### 3.2 알림 아이템

| 요소명 | 유형 | 설명 |
|--------|------|------|
| 알림 아이콘 | Icon (color-coded) | 바우처: 노랑, 실적: 파랑, 그룹: 초록 |
| 알림 제목 | Text (bold if unread) | "라운지 이용권이 3일 후 만료됩니다" |
| 알림 본문 | Text (muted) | 상세 내용 |
| 시간 | Text (muted) | "2시간 전", "어제" 등 (relative time) |
| 액션 버튼 | Button (optional) | 그룹 초대: "수락" / "거절" |
| 읽음 표시 | Dot (blue) | 미읽은 알림에만 표시 |

### 3.3 알림 설정 화면

- **진입 경로**: 설정 → 알림 설정, `/settings/notifications`

| 요소명 | 유형 | 기본값 |
|--------|------|--------|
| 바우처 만료 알림 | Switch | ON |
| 실적 변동 알림 | Switch | ON |
| 실적 리마인더 | Switch | ON |
| 그룹 초대 알림 | Switch | ON |
| 그룹 활동 알림 | Switch | ON |

## 4. Acceptance Criteria

| # | Given | When | Then |
|---|-------|------|------|
| AC-01 | 바우처 만료 D-7일에 | 배치 작업이 실행되면 | 해당 사용자에게 만료 임박 알림이 생성된다 |
| AC-02 | 실적이 2구간에서 3구간으로 변경되면 | 실적 집계 후 | "축하합니다! 3구간 달성" 알림이 생성된다 |
| AC-03 | 월말 D-7에 | 실적 미달 카드가 있으면 | "이번 달 목표까지 XX원 남았습니다" 리마인더가 생성된다 |
| AC-04 | 그룹 초대를 받으면 | 알림이 생성되고 | "수락"/"거절" 액션 버튼이 표시된다 |
| AC-05 | 그룹 초대 알림에서 "수락"을 누르면 | 그룹에 멤버로 추가되고 | 알림이 읽음 처리된다 |
| AC-06 | 알림 목록에서 알림을 클릭하면 | 해당 알림이 읽음 처리되고 | 관련 화면으로 이동한다 |
| AC-07 | "모두 읽음 처리"를 누르면 | 전체 미읽은 알림이 | 읽음 상태로 변경된다 |
| AC-08 | 알림 설정에서 "바우처 만료 알림"을 OFF하면 | 이후 바우처 만료 알림이 | 생성되지 않는다 |

## 5. Edge Cases & 에러 시나리오

| 시나리오 | 조건 | 처리 |
|----------|------|------|
| 대량 알림 | 한 번에 알림 50개 이상 생성 | 배치 처리, 목록은 페이지네이션 |
| 알림 삭제 | 오래된 알림 정리 | 90일 이후 자동 삭제 (배치) |
| 동시 읽음 처리 | 다중 디바이스에서 동시 접근 | 멱등성 보장 (이미 읽음이면 무시) |
| 그룹 초대 만료 | 초대가 만료된 후 알림의 "수락" 클릭 | "초대가 만료되었습니다" 에러 |
| 탈퇴한 그룹 알림 | 이미 탈퇴한 그룹의 활동 알림 | 알림 생성 시 멤버십 확인, 비멤버 제외 |
| 알림 설정 미존재 | 신규 가입 시 알림 설정이 없는 경우 | 모든 알림 ON (기본값) |

## 6. API 연동

| 화면/기능 | Method | Endpoint | 비고 |
|-----------|--------|----------|------|
| 알림 목록 | GET | `/api/v1/notifications?type={type}` | 커서 페이지네이션 |
| 미읽은 수 | GET | `/api/v1/notifications/unread-count` | 배지 표시용 |
| 읽음 처리 | PATCH | `/api/v1/notifications/{id}/read` | |
| 전체 읽음 | PATCH | `/api/v1/notifications/read-all` | |
| 알림 설정 조회 | GET | `/api/v1/notification-preferences` | |
| 알림 설정 변경 | PATCH | `/api/v1/notification-preferences` | 유형별 on/off |
| 그룹 초대 수락 | POST | `/api/v1/groups/{groupId}/invitations/{id}/accept` | 알림 액션 |
| 그룹 초대 거절 | POST | `/api/v1/groups/{groupId}/invitations/{id}/decline` | 알림 액션 |

## 7. 데이터 모델 연동

### 관련 테이블

| 테이블 | 역할 |
|--------|------|
| `notification` | 알림 레코드 (type, title, body, read_at, action_url) |
| `notification_preference` | 사용자별 알림 설정 (type별 on/off) |
| `card_voucher` / `user_voucher` | 바우처 만료 알림 소스 |
| `user_performance` | 실적 변동 알림 소스 |
| `group_invitation` | 그룹 초대 알림 소스 |

### 알림 생성 이벤트 매핑

| 도메인 이벤트 | 알림 유형 | 생성 조건 |
|--------------|----------|----------|
| VoucherExpiringEvent | VOUCHER_EXPIRY | D-7, D-3, D-1 배치 |
| PerformanceTierChangedEvent | PERFORMANCE_CHANGE | 구간 상승/하락 시 |
| MonthEndReminderEvent | PERFORMANCE_REMINDER | 월말 D-7 배치, 미달 카드 존재 시 |
| InvitationSentEvent | GROUP_INVITE | 초대 생성 시 |
| PaymentCreatedEvent (group) | GROUP_ACTIVITY | 그룹 결제 등록 시 (본인 제외) |
