# CardWise UX & 디자인 아키텍처 (Design Architecture)

이 문서는 CardWise의 시각적 정체성, 사용자 경험 원칙, 그리고 디자인에서 퍼블리싱까지의 연결 구조를 정의합니다.

## 1. 디자인 원칙 (Design Principles)

### 1.1 톤앤매너 (Tone & Manner)
- **Rose Minimal**: 세련되고 미니멀한 로즈/핑크 톤을 베이스로 하되, 전문적인 "핀테크" 감성을 유지합니다.
- **입체감과 깊이 (Depth & 3D)**: 플랫한 디자인보다는 소프트 섀도우, 3D 레이어링, 그라데이션을 활용하여 매끄러운 깊이감을 강조합니다.
- **가독성 최우선 (Readability)**: 
    - ⚠️ **핵심**: 핑크색 배경 위에 **흰색 글자**를 사용하는 것을 지양합니다. (배경과 글자의 대비가 낮아 가독성이 떨어지는 현상 방지)
    - **해결**: 핑크 배경에서는 고대비 다크 그래파이트(#1f2937) 또는 딥 로즈(#881337) 폰트를 활용하여 시각적 명확성을 확보합니다. 흰색 글자는 어두운 배경이나 충분한 대비가 확보된 곳에서만 사용합니다.

### 1.2 인터랙션 가이드
- **Micro-interactions**: 버튼 클릭 시 `active:scale-95`, 리스트 스크롤 시 페이드 인 효과 등을 적용합니다.
- **워드클라우드 (Word Cloud)**: 인사이트 메뉴에서 사용자의 태그를 시각화할 때 동적 워드클라우드를 사용합니다.
- **실험적 차트 (Experimental Charts)**: 3D 버킷, 입체적 게이지, 애니메이션 영역 차트 등을 도입합니다. (기존 차트 기능 유지)

## 2. 모바일-웹 통합 로드맵 (Platform Parity)

### 2.1 라우팅 체계
- **Mobile**: `/mobile/*` (하단 내비게이션 중심)
- **Web**: `/web/*` (사이드바 중심 데스크톱 레이아웃)
- **OPS**: `/ops/*` (개발 전입 운영 도구 - 기존 `/ops/live` 고도화)

### 2.2 공통 컴포넌트 (`src/components/mobile`)
- 차트, 모달, 네비게이션 소스를 공유하되 레이아웃에 맞춰 반응형으로 작동하도록 설계합니다.

---

## 3. 디자인-퍼블리시-프론트-백엔드 연결 구조

1.  **Design**: .pen 파일을 통한 UI 설계 및 가이브라인 수립.
2.  **Publishing**: Tailwind CSS 기반의 `index.css` 및 글로벌 테마 변수(`variables`) 적용.
3.  **Frontend**: Next.js 기반의 컴포넌트화 및 라우팅.
4.  **Backend**: REST API를 통한 데이터 연동 (현재는 Mock Data 위주, 점진적 API 연동).
