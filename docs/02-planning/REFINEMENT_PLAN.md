# CardWise v4.2 고도화 및 품질 관리 계획

## 1. 개요

현재 CardWise 프로젝트는 프리미엄 UI가 구현되었으나, 탭 전환 시 상호작용 오류, 데이터 불일치, 세션 관리 기능의 미작동 등 초기 안정성 문제가 발견되었습니다. 본 문서는 이를 해결하기 위한 리팩토링 및 누락 기능 구현 계획을 기술합니다.

## 2. 긴급 조치 사항 (완료)

- [x] **탭 ID 불일치 해결**: `dashboard`와 `home`으로 혼용되던 탭 식별자를 `home`으로 통일하여 홈 화면 증발 및 영문 타이틀 노출 버그 해결.
- [x] **세션 타임아웃 로직 고도화**: `alert`에 의한 중단 현상을 방지하고 `window.location.href` 및 `useEffect` 기반의 전역 감시 시스템으로 재구축.
- [x] **소스 코드 대규모 리팩토링**: `page.tsx`의 비대화(500라인 이상)를 해결하기 위해 `HomeView`, `CardsView`, `LedgerView` 등 도메인별 뷰 컴포넌트로 분리.

## 4. 시니어 아키텍트 추천 기능 (Missing Features)

1. **데이터 영속성 레이어**: 현재는 새로고침 시 모든 데이터가 초기화됩니다. `localStorage` 또는 `TanStack Query` 기반의 로컬 캐시 시스템 도입을 추천합니다.
2. **동적 라우팅 고도화**: `/mobile/card-detail` 등의 하위 페이지를 실제 라우트로 분리하여 브라우저 '뒤로가기'와 시스템 가스테이트 일치.
3. **PWA (Progressive Web App)**: 모바일 앱의 느낌을 주기 위해 `manifest.json` 및 `Service Worker`를 통한 오프라인 지원 및 설치 유도.
4. **글로벌 테마 엔진**: 현재 하드코딩된 색상값들을 CSS 변수로 치환하여 '다크 모드' 및 '카드 위 아이덴티티 반영 테마' 지원.

## 5. 개발 철학 반영

- **KISS (Keep It Simple, Stupid)**: 복잡한 상태 관리 라이브러리 대신 Context API와 URL Params 기반의 가벼운 상태 유지.
- **Atomic Design**: 이미 컴포넌트가 잘 분리되어 있으나, UI 프리미티브(Button, Card 등)의 일관성 강화.
