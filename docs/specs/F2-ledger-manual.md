# F2 - 가계부 (수동 입력 + 해외결제)

> 최종 갱신: 2026-03-18

---

## 1. 개요

### 목적

사용자가 결제 내역을 수동으로 입력하여 가계부를 관리하는 기능을 정의한다. 1건의 결제(Payment)에 N개의 품목(PaymentItem)을 입력할 수 있으며, 각 품목별로 카테고리, 혜택 적용, 태그 부여가 가능하다. 쿠팡, 마켓컬리 등 복수 품목 주문에 대응한다.

다중 통화를 지원하여 Amazon, AliExpress 등 해외 결제와 해외 여행 시 현지 통화 결제를 원본 통화와 원화(KRW) 환산 금액 모두 기록한다. 그룹 가계부 결제 입력 시 그룹을 선택하여 공유 가계부에 기록할 수 있다.

### 대상 사용자

- 일상 결제를 기록하고 관리하려는 회원
- Amazon, AliExpress 등 해외 쇼핑몰에서 결제하는 회원
- 해외 여행 중 현지 통화로 결제하는 회원
- 가족/그룹 공유 가계부에 결제를 기록하려는 그룹 멤버

## 3. 화면 명세

### 3.1 결제 입력 화면

- **진입 경로**: 홈 → FAB "+" 버튼, `/payments/new`

| 요소명 | 유형 | 필수 | 검증 규칙 |
|--------|------|------|----------|
| 결제일 | DatePicker | ✅ | 기본값: 오늘, 미래 날짜 불가 |
| 카드 선택 | Select | ✅ | 등록된 카드 목록, 주 사용 카드 기본 선택 |
| 가맹점명 | Input | ✅ | 1~100자 |
| 통화 선택 | Select | ✅ | 기본값: KRW, 10개 통화 지원 |
| 총 결제 금액 | NumberInput | ✅ | 1원 이상, 소수점 2자리 (외화) |
| 원화 환산 금액 | NumberInput (readonly) | 조건부 | 외화 선택 시 자동 계산, 수동 수정 가능 |
| 환율 | NumberInput | 조건부 | 외화 선택 시 표시, 최신 환율 자동 입력 |
| 그룹 선택 | Select | ❌ | "개인" (기본) 또는 소속 그룹 목록 |
| 메모 | Textarea | ❌ | 최대 500자 |
| 품목 추가 버튼 | Button | - | "품목 추가" |
| 저장 버튼 | Button (primary) | - | 폼 검증 통과 시 활성화 |

### 3.2 품목 입력 (결제 입력 화면 내)

| 요소명 | 유형 | 필수 | 검증 규칙 |
|--------|------|------|----------|
| 품목명 | Input | ✅ | 1~100자 |
| 금액 | NumberInput | ✅ | 1원 이상, 합계 = 총 결제 금액 검증 |
| 카테고리 | Select | ✅ | 시스템 카테고리 목록 |
| 태그 | Combobox (multi) | ❌ | 자동완성, 최대 10개 |
| 적용 혜택 | Select | ❌ | 선택된 카드의 혜택 목록 (해당 카테고리 필터) |
| 삭제 | IconButton (trash) | - | 품목 1개 이상 유지 |

**동작 규칙**:
- 품목이 1개일 때는 총 결제 금액과 동일하게 자동 입력
- 품목 2개 이상일 때 합계 ≠ 총 결제 금액이면 경고 표시
- 품목 삭제 시 최소 1개는 유지

## 4. Acceptance Criteria

| # | Given | When | Then |
|---|-------|------|------|
| AC-01 | 결제 입력 화면에서 | 카드, 가맹점, 금액, 카테고리를 입력하고 저장하면 | payment + payment_item 레코드가 생성된다 |
| AC-02 | 품목 2개를 입력했을 때 | 품목 금액 합계가 총 결제 금액과 다르면 | 경고 메시지가 표시된다 |
| AC-03 | 통화를 USD로 변경하면 | 환율 필드가 표시되고 | 최신 환율이 자동 입력된다 |
| AC-04 | USD 50을 입력하고 환율 1,300이면 | 원화 환산 금액이 | 65,000원으로 자동 계산된다 |
| AC-05 | 결제 저장 시 | exchange_rate_snapshot에 해당 환율이 없으면 | 새 스냅샷이 생성된다 |
| AC-06 | 결제 목록에서 기간 필터를 변경하면 | 해당 기간의 결제만 표시되고 | 월 합계가 업데이트된다 |
| AC-07 | 결제 상세에서 수정 버튼을 누르면 | 편집 모드로 전환되고 | 품목 추가/삭제/수정이 가능하다 |
| AC-08 | 결제를 삭제하면 | 관련 payment_item, payment_item_tag가 | CASCADE 삭제된다 |
| AC-09 | 그룹을 선택하고 결제를 저장하면 | payment.group_id가 설정되고 | 그룹 가계부에 표시된다 |
| AC-10 | 주 사용 카드가 설정된 상태에서 | 결제 입력 화면 진입 시 | 해당 카드가 기본 선택된다 |

## 5. Edge Cases & 에러 시나리오

| 시나리오 | 조건 | 처리 |
|----------|------|------|
| 등록된 카드 없음 | 카드 0장인 상태에서 결제 입력 시도 | "카드를 먼저 등록해주세요" + 카드 등록 링크 |
| 환율 조회 실패 | 외부 환율 API 장애 | 마지막 캐시된 환율 사용 + "최신 환율이 아닐 수 있습니다" 안내 |
| 품목 합계 불일치 | 품목 합 ≠ 총 금액 | 경고 표시, 저장은 허용 (실제 카드 청구와 차이 가능) |
| 미래 날짜 결제 | 내일 이후 날짜 선택 | 클라이언트 검증 → 오늘 이전 날짜만 허용 |
| 대량 품목 | 품목 20개 이상 | 최대 50개 제한, 스크롤 가능 |
| 0원 결제 | 금액 0 입력 | 검증 실패 → "1원 이상 입력해주세요" |
| 그룹 탈퇴 후 결제 | 이미 탈퇴한 그룹의 결제 조회 | 조회 가능 (읽기 전용), 수정/삭제 불가 |
| 네트워크 오류 | 저장 중 연결 끊김 | 로컬 임시 저장 → 재연결 시 재시도 안내 |

## 7. 데이터 모델 연동

### 관련 테이블

| 테이블 | 역할 |
|--------|------|
| `payment` | 결제 건 (카드, 가맹점, 총액, 통화, 그룹) |
| `payment_item` | 결제 품목 (카테고리, 금액, 혜택) |
| `payment_item_tag` | 품목-태그 M:N |
| `user_card` | 사용자 등록 카드 |
| `exchange_rate_snapshot` | 환율 스냅샷 |
| `category` | 카테고리 마스터 |
| `card_benefit` | 카드 혜택 (적용 혜택 선택용) |
| `ledger_group` | 그룹 (그룹 결제 시) |

### 핵심 비즈니스 로직

1. **결제 저장 시**: payment 생성 → payment_item N개 생성 → payment_item_tag 생성 → 실적 자동 갱신 (PaymentCreatedEvent)
2. **다중 통화**: `currency ≠ KRW`이면 original_amount에 원본 금액, krw_amount에 환산 금액 저장. 실적 계산은 항상 krw_amount 기준
3. **그룹 결제**: payment.group_id 설정 → 그룹 통계에 반영, 개인 통계에도 반영
